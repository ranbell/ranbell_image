## 1. 基本的な考え方 — 画像を意味の座標として扱う

Ranbell Image に追加されたすべての画像は、**高次元の埋め込みベクトル**（768次元の数値列）に変換されます。このベクトルは構図・カラーパレット・スタイル・ムード・被写体・雰囲気といった視覚的な意味を1本の数学的な対象に圧縮したものです。

この変換によって、画像は**意味空間上の座標**として扱えるようになります。似た世界観の画像は近くに配置され、全く異なる画像は遠くに配置されます。

この発想がインスパイアモードの根幹です。「このファイル名の画像はどれか」や「このタグが付いた画像はどれか」ではなく、次のような問いに答えられます：

| 問い | 数学的な意味 |
|---|---|
| 「この画像に似ているものを探して」 | 近い座標を探す |
| 「概念 A を足して B を引いた結果に近いものは」 | A + B − C を計算し最近傍を探す |
| 「A と B の中間にある画像を見せて」 | A から B への線形補間 |
| 「この画像の正反対は」 | ベクトルを符号反転し最近傍を探す |
| 「コレクション中で最も孤立した画像は」 | 他のすべてから最も遠い座標を探す |

状況に応じて、画像を3種類の方法で「読み」ます：

- **ベクトルとして** — 全情報が圧縮された埋め込み
- **タグとして**（WD14） — Danbooru カテゴリラベル。視覚的な語彙を言語で表現したもの
- **座標として**（UMAP） — コレクション全体を2Dに投影したときの位置情報

### 複雑度の分類

```

  ├── 錬金術     — ベクトル加減算のみ

  ├── セレンディピティ — パーセンタイル帯からのランダムサンプリング
  └── アウトライアー   — 符号反転または密度分析

  ├── ディスカバー    — DiscoverQuery
  └── グループ検索   — GroupBy + テキスト埋め込み

LLM/VLM 統合（低速・非決定論的・高表現力）
  ├── アノマリー     — LLM タグ生成（軽量）
  ├── ブレスト       — LLM 文章生成（中量）
  └── インバージョン — VLM 3ステージパイプライン（最重量）
```

## 3. 各モードリファレンス

---

### セレンディピティ ✨

**できること：** 参照画像に「似ているが、似すぎていない」画像を発見する。「よく知っている」と「全く関係ない」の中間地帯を探索する。

**渡すもの：** 参照画像 1〜6 枚。

**仕組み：**

参照画像の埋め込みベクトルを平均化してクエリベクトルを作成。Qdrant がコサイン類似度で上位 1,000 件を取得してスコア順に並べます。次に、そのスコアの第 25 パーセンタイル（P25）と第 75 パーセンタイル（P75）を計算し、P25〜P75 の帯域内からランダムにサンプリングします。

このパーセンタイル方式は、使用している埋め込みモデルのスコア分布に自動的に適応します。固定閾値（例: 0.5〜0.7）では、モデルによってスコア分布が大きく異なるため有効な結果が得られないことがありますが、パーセンタイル帯はコレクションの**実際の分布**に常にアンカーされます。

**向いている用途：** マンネリ打破。存在を忘れていた画像の発掘。いつもと違う角度からのアイデア出し。

---

### 錬金術 ⚗️

**できること：** 「この画像の色合い＋あの画像の構図−都会っぽさ」のように、視覚的な概念を実際のベクトル演算で足し引きする。

**渡すもの：** 加算画像（最大 3 枚）と減算画像（最大 3 枚）。

**仕組み：**

各画像の埋め込みベクトルを加算・減算する前に、個別に単位長へ正規化します。正規化は最初だけでなく加算のたびに行います（**反復正規化**）。これにより、特定の画像の magnitude が大きくても、その画像が結果を支配することを防ぎます。

得られた合成ベクトルは、指定した概念の組み合わせを満たす意味空間の領域を指します。Qdrant がその領域に最も近い実在画像を検索します。

**核心：** 画像が座標になった瞬間、ベクトルの加減算は現実の操作になります。「アウトドアシーン」を「ファッション写真」から引くと、クエリが実際にアウトドアのクラスターから離れてインドアファッションのクラスターへ移動します。

**向いている用途：** 言葉では説明しにくい創作意図の表現。「これに似た感じで、あれを少し加えて、でもこれは外したい」というニュアンスの探索。

---

### モーフ 🌊

**できること：** 2つの画像の間にある自然なグラデーションを5段階で見る。A から B への概念移行を段階的に可視化する。

**渡すもの：** 画像 A と画像 B の 2 枚。

**仕組み：**

2つの埋め込みベクトル間で、t = 0.2, 0.4, 0.6, 0.8 の 5 段階に線形補間（LERP）を行います。各ステップで補間ベクトルを正規化してから Qdrant で最近傍を検索し、各ステップ 4 枚の代表画像を取得します。

**向いている用途：** 移行の美学。昼→夜、静か→にぎやか、リアル→スタイライズド。全く異なる 2 つの画像の視覚的な「橋渡し」を探すとき。

---

### アノマリー ⚡

**できること：** 参照画像の文脈を理解したうえで、そのコレクションにとって「珍しいが文脈的に意味のある」要素を注入した画像を発見する。ランダムな驚きではなく、根拠のある意外性。

**渡すもの：** 参照画像 1〜6 枚。

**仕組み：**

参照画像から WD14 タグ上位 30 件を収集して頻度をカウント。この高頻度タグのリストを LLM（Ollama）に渡し、「このコンテキストでは稀少だが意味的に自然な Danbooru タグを 3 つ提案して」と依頼します。

LLM が提案したアノマリータグを元のタグに追加し、テキストとして埋め込んでベクトル検索に使用します。レスポンスには「LLM が注入したタグ」が含まれるため、何が驚きの源泉だったかを確認できます。

**向いている用途：** いつもとは違う組み合わせの発見。構図のマンネリ打破。「文脈的には合っているのに意外性がある」画像との出会い。

---

### インバージョン 🪞

**できること：** 参照画像の世界観・雰囲気を 5 つの意味軸に沿って意味論的に反転させ、「正反対の世界」を体現する画像をコレクションの中から探す。

**渡すもの：** 参照画像 1〜3 枚。

**5つの反転軸：**

| 軸 | 反転される要素 |
|---|---|
| **Visual** | 明暗・寒暖・密度・内外 |
| **Mood** | 穏やか↔混沌・幸福↔憂鬱・活動的↔静寂 |
| **Subject** | 年齢・表情・姿勢・人数 |
| **Style** | 細密↔ミニマル・彩度変化 |
| **Narrative** | 日常↔幻想・親しみ↔異質 |

**制約：** キャラクターの外見的特徴（髪色・瞳・固有人物など）は反転しません。「世界」が変わり、「キャラクター」は変わらないのが設計の意図です。

**仕組み — VLM 3ステージパイプライン（SSE ストリーミング）：**

最も計算コストが高いモードです。3回の VLM 処理が順番に実行され、結果は Server-Sent Events でリアルタイムにブラウザへストリーミングされます。

**Stage 1 — 生成：** VLM が参照画像を解析し、以下を生成します：
- 反転世界の物語（300〜500 語）
- Danbooru/WD14 タグ（100〜150 個）
- 自然言語プロンプト（150〜200 語）
- ネガティブタグ（元のシーンの要素を排除する 20〜40 個）

**Stage 2 — バリデーション：** 2回目の VLM パスが Stage 1 の出力を検査し、一貫性と安全性の問題点をリストアップします。

**Stage 3 — リファインメント**（問題検出時のみ）：3回目の VLM パスがバリデーションのフィードバックを基に出力を改善します。

最終的なタグとプロンプトを埋め込んで Qdrant で検索します。

**3ステージが必要な理由：** VLM の出力は、検証なしでは自己矛盾を含むことがあります（例: 「室内」に反転したのに屋外特有のタグが残る）。自己修正ループにより、人間が確認しなくても一貫した品質が得られます。

**向いている用途：** 「この画像の世界が全く違ったら？」という問い。創作の方向性探索。すでに好きな画像の「対極」をコレクションの中から発見する。

---

### ディスカバー 🧭

**できること：** 「このターゲットに近く、こういう方向性に寄っていて、あの方向は避けたい」という複合的な意図を1回のクエリで表現して探索する。

**渡すもの：** ターゲット画像 1 枚 + 方向を示すポジティブ/ネガティブ画像ペア。

**仕組み：**

Qdrant の `DiscoverQuery` を使用します。通常の類似検索が「X に最も近いものを探す」という最適化を行うのに対し、DiscoverQuery は「X に近く、かつ P に似て N に似ていない」という複合最適化を行います。

ポジティブ/ネガティブのペアを追加するたびに、探索方向への圧力が加わります。

**向いている用途：** 制御された方向性探索。「近いけど、あっちの方向に寄せて、この要素は避けて」という漠然とした探索意図を画像で指し示したいとき。

---

### グループ検索 🗂️

**できること：** 自然言語クエリでセマンティック検索しながら、結果を生成モデル・ファイル形式などでグループ化する。同じ概念を複数のモデルがどう解釈したかを横断比較できる。

**渡すもの：** テキストクエリ。

**仕組み：**

クエリテキストを Ollama で埋め込んでベクトル化し、Qdrant の `GroupBy API` で検索します。結果はモデル名・拡張子などでグループ分けされ、グループごとに画像が返されます（通常: 1グループあたり 3 枚、最大 10 グループ）。

**向いている用途：** モデル間比較。「夕暮れの海岸」というクエリに対して、DreamShaper・SDXL Turbo・Anima がそれぞれどんな表現をしたかを一覧する。次の生成でどのモデルを選ぶかの判断材料にする。

---

### ブレンド ⚖️

**できること：** 複数の画像に −1.0〜+1.0 の重みを割り当て、視覚的な概念を割合を指定してミキシングする。

**渡すもの：** 画像 2〜4 枚、それぞれウェイトスライダー付き。

**仕組み：**

各画像の埋め込みを単位長に正規化してからウェイトでスケーリングし、すべてを合算。最終的な合計ベクトルを正規化して Qdrant で検索します。重みが 0.5 を超える画像（「強く含める」と指定した画像自身）は結果から除外されます。

**錬金術との違い：**
- 錬金術: +1/−1 の二値ロール、反復正規化による magnitude 管理
- ブレンド: 連続値ウェイト（−1.0〜+1.0）、加重線形結合

明確な「足す/引く」操作なら錬金術。「A を 60%、B を 30%、C を 10%」という比率指定なら ブレンド。

**向いている用途：** 細かいムード調整。複数の参照画像から狙ったターゲット美意識を比率を制御しながら合成する。

---

### アウトライアー 🌌

**できること：** コレクション全体の「意味的な極地」や「孤立した島」を発見する。参照画像は不要 — コレクション自体の性質を探索するモード。

**渡すもの：** なし（参照画像不要）。

**2つのサブモード：**

**アンチポード（数学的な対極）：**

コレクション全画像の埋め込みを1つの重心ベクトルに平均化します。この重心は「コレクションの典型的な画像」を表します。この重心ベクトルの符号を反転（全要素に −1 を掛ける）すると、意味空間の「対極」を指すベクトルが得られます。この対極点に最も近い画像が、コレクションの重心から最も遠い画像 — つまり最も「外れ値」的な画像です。

**孤立島（密度ベース）：**

Qdrant ペイロードに保存された UMAP 2D 座標を使って、各画像の局所密度を計算します。半径 2.0 ユニット以内の隣接画像数をカウントし、この数が最も少ない（周囲に似た画像がいない）画像群を孤立点として特定。その中からランダムにサンプリングします。

フォールバック: UMAP 座標が未計算の場合はランダムサンプリングを使用。

**向いている用途：** コレクション中で最もユニーク・実験的な作品の発掘。どのクラスターにも属さない一点物の発見。コレクションの「主流」に対して際立った個性を持つ作品との出会い。

## 4. ブレスト 💡

**できること：** 発見した画像群を素材として LLM に渡し、3〜5 つのシーンアイデアを生成してもらう。そのまま錬成スタジオへ送信できる。

**渡すもの：** インスパイアモードで発見した画像（ブレスト入力として選択）。アノマリーやインバージョンを使用した場合、そこで生成されたタグも自動的に統合されます。

**仕組み：**

選択画像から WD14 タグを収集し、アノマリー/インバージョンの追加タグと統合。この語彙一式を Ollama に渡して、「これらのタグから魅力的なシーンアイデアを 3〜5 つ、Markdown 形式で提案して」と依頼します。

レスポンスは Server-Sent Events でリアルタイムにストリーミング。提案がリアルタイムで形成されるのを目で追えます。各シーン提案は個別に「錬成スタジオへ送信」できます。

**対応言語：** 日本語・英語。

**向いている用途：** 「面白い画像を見つけたけど次に何を作ればいいかわからない」というギャップを埋める。視覚的なインスピレーションを具体的な創作指示に変換する橋渡し。

## 5. クリエイティブパイプライン全体像

インスパイアモードとブレストは独立したツールではなく、発見から生成まで繋がったパイプラインを形成しています：

```
発見（Discovery）
  └── セレンディピティ / アノマリー / インバージョン / アウトライアー / ...
      意識していなかった画像を見つける

        ↓

分析（Analysis）— 自動
  └── 結果画像から WD14 タグを収集
      アノマリー / インバージョンのタグを統合
      視覚的特徴を言語の語彙に変換

        ↓

発想（Ideation）
  └── ブレスト
      LLM がタグ語彙から 3〜5 つのシーン提案を生成
      リアルタイムストリーミング

        ↓

錬成（Refinement）
  └── 錬成スタジオ
      参照画像 + ブレスト提案 → VLM が完全なプロンプトを生成
      出力スタイル: Danbooru / 自然言語 / ハイブリッド

        ↓

生成（Generation）
  └── ComfyUI（ワンクリック送信）
      生成した画像をコレクションに追加
      コレクションが豊かになる → 次の探索の精度が上がる
```

生成された画像はコレクションに還流されてインデックスに追加されます。コレクションが大きくなるほど意味空間の解像度が上がり、探索の精度と驚きの質が向上します。

## 6. ベクトル演算の仕組み

### 正規化がなぜ重要か

すべての埋め込みベクトルは比較前に単位長（L2ノルム = 1）に正規化されます。これにより、類似度が**方向**（高次元空間内のベクトル間の角度）で測定され、magnitude に左右されません。正規化なしでは、magnitude が大きいベクトルが加算操作を支配してしまいます。

### 反復正規化（錬金術・モーフ）

複数のベクトルを加算するモードでは、加算のたびに再正規化を行います。これにより「magnitude ドリフト」を防ぎます。単純加算を繰り返すと合計ベクトルの magnitude が大きくなり、後から追加されるベクトルの相対的な影響力が小さくなっていくためです。毎回の正規化により、すべての画像が**等しい発言権**を持ちます。

### コサイン類似度とドット積の等価性

Qdrant は正規化済みベクトルに対してドット積で検索します。クエリベクトルも格納ベクトルも単位長に揃えているため、ドット積はコサイン類似度と等価になります。つまり、意味的な角度距離で最も近い画像が先に返されます。

### MRL — 2フェーズ検索

Ranbell Image はすべての Qdrant 検索に Matryoshka Representation Learning（MRL）を使用しています。各画像の埋め込みには完全な 768 次元ベクトルとともに、256 次元のコンパクトなプレフィックスが保存されています。検索は 2 段階で実行されます：

1. **プリフェッチ** — 256 次元ベクトルでコレクション全体を高速スキャンし、候補セットを絞り込む
2. **リランク** — 完全な 768 次元ベクトルで候補を精密にスコアリングする

この 2 段階設計により、大規模コレクションでも精度を落とさずに高速な検索を実現しています。
